WELCOME TO OUR ALL PROJECT

A LITTLE BIT ABOUT OUR PROJECT:

One of our goals is to register an entire class to a lesson through the use of NFC cards. This will be done without any student needing to be present. To achieve this we will use: a Beaglebone Black and an RFID chip reader and writer.

In addition to the previous aim, we want to emulate or create a master card that is able to open any room in the Engineering and computing Building.

SOME PROBLEMS WE RAN INTO:

  • One problem we encountered was that the documentation the card reader had was in Chinease, or very broken English, so translating this slowed progress as there was very limited usefull information/ instructions online for this machine.
  • The NFC chip had basic features so we had to figure out a way too work around this as the libraries were not working.
  • The beaglebone deletes imporant libraries necessary to make the reader work. This happens when it reaches full storage. A way to fix this would be to use a rasberry pie or to use an SD card.
  • Having multiple people working on the front end and the back end of the website was tricky when it came to implementing the two together. This was because different variable names were used, in addition to this, two different languages were used, so combining these came with their own problems.
  • The card reader we were using changed some meta-data on the cards which affected the uid displayed to us. This meant that we were having problems reading data off cards and putting data onto blank ones. One way we solved this was using better equipment.

WHAT WE HAVE LEARNED:

Throughout this project there have been many challenges we have come across and had to solve. These challegnges in turn were a learning curve for all of us. One of the many lessons we learned was that different types of NFC cards have different levels of security/ encryption. This meant that getting the information we wnated out of them was more difficult than intended. We also learn't that the inital way of going about this project broke some of the rules of the RIPA act. The cards are mifaree classic. We found out this information from the first part of coummunication that releases manufactuerer details and the UID which arent encrypted. The card is Type A (According to the ISO 14443 standard for NFC's) What descries and structures the intial communication between the card and the reader allowing further communication lke in TCP, From this point the mifaree classic card differs from this standard by encrypting any following data The cards us a propiatory encryption system called crypto-1 which uses random generated numbers to validate the reader. This is done when the card want access to a section of the card by sending the reader a nonce which it has to send back encrypted with crypto-1 encryption using the secret key
