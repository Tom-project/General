Explanation

  • The back end of our website which is used to communicate with the webserver uses jquery (a programming language). It sends queries to the web server through an api, this was developed by Aaron

    Depending on what api request was sent, the webserver will return with different things

    • However it will always returns a message, either if the request was succesful or unsuccesful
    • With a succesfull request it will return the id of the card from the server
    • As well as this it will return an object, “data”, containing information about the card such as UID etc.

    Functions: getScan

    • When you click the "Scan" button on the webpage it will run the function written in Jquery which will make a POST request to the server containing all the information about the most recently scanned card
    • The server then adds the card with info to the list of available cards, the list can be updated using the "List" button.

    Functions: getList

    • When you click the "List" button on the webpage it will run the function written in Jquery which will make a POST request to the server with no data.
    • The Server returns the list of cards stored with all their information. This can be used to update the list.
    • It will also iterate over each item in the list, creating a span object for each one that is displayed to the user.
    • Amazingly formatted by Tom and Brandon’s css.

    Planned features: Emulate cards

    • Our goal is to make a POST request to the server, again to get all the info of the specified card
    • And then for it to communicate with the BeagleBone to tell it to emulate the selected card.
    • Allowing us to send the data to a blank card.
